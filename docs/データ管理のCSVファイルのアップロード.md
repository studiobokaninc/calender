# データ管理のCSVファイルのアップロード

## 概要
このドキュメントでは、プロジェクト管理システムにおけるCSVファイルを使用したデータインポート機能について説明します。

## CSVファイルの仕様

### ファイル形式
- 文字コード: UTF-8
- 区切り文字: カンマ (,)
- 改行コード: LF (Unix形式)

### ファイル構造
CSVファイルは以下の2つのセクションで構成されます：

1. プロジェクト情報セクション
2. タスク情報セクション

### プロジェクト情報セクション
```
プロジェクト情報
プロジェクト名,開始日,終了日,説明
プロジェクトX,2024/03/01,2024/03/31,プロジェクトXの説明
```

#### フィールド説明
| フィールド | 説明 | 必須 | 形式 |
|------------|------|------|------|
| プロジェクト名 | プロジェクトの名称 | ○ | 文字列（1-100文字） |
| 開始日 | プロジェクトの開始日 | - | YYYY/MM/DD |
| 終了日 | プロジェクトの終了日 | - | YYYY/MM/DD |
| 説明 | プロジェクトの説明 | - | 文字列 |

### タスク情報セクション
```
タスク情報
タスク名,期日,説明,担当者,コスト,タイプ,seqID,shotID,依存タスク
T1,2024/03/15,T1の説明,user1,16,fx,SEQ001,SHOT001,
T2,2024/03/20,T2の説明,user2,24,animation,SEQ001,SHOT002,T1
T3,2024/03/25,T3の説明,user3,32,comp,SEQ002,SHOT001,"T1,T2"
```

#### フィールド説明
| フィールド | 説明 | 必須 | 形式 |
|------------|------|------|------|
| タスク名 | タスクの名称 | ○ | 文字列 |
| 期日 | タスクの期限 | - | YYYY/MM/DD |
| 説明 | タスクの説明 | - | 文字列 |
| 担当者 | タスクの担当者 | - | ユーザー名 |
| コスト | タスクの工数 | - | 数値 |
| タイプ | タスクの種類 | - | 文字列 |
| seqID | シーケンスID | - | 文字列 |
| shotID | ショットID | - | 文字列 |
| 依存タスク | 依存するタスク名 | - | カンマ区切りのタスク名（複数指定時は引用符で囲む） |

### 注意事項
1. セクション名（「プロジェクト情報」「タスク情報」）は必ず記述してください
2. 各セクションのヘッダー行は必ず記述してください
3. 日付は「YYYY/MM/DD」形式で記述してください
4. 複数の依存タスクを指定する場合は、引用符（"）で囲んでください
   - 例: `"T1,T2"`
5. 空のフィールドは省略可能です
6. プロジェクト名とタスク名は重複しないようにしてください（自動的に連番が付与されます）
7. プロジェクト名は1文字以上100文字以下である必要があります
8. タスクの優先度が指定されていない場合は、デフォルトで「MEDIUM」が設定されます
9. タスクのステータスは自動的に「TODO」に設定され、ステータス履歴が作成されます

### サンプルCSV
```
プロジェクト情報
プロジェクト名,開始日,終了日,説明
プロジェクトX,2024/03/01,2024/03/31,プロジェクトXの説明

タスク情報
タスク名,期日,説明,担当者,コスト,タイプ,seqID,shotID,依存タスク
T1,2024/03/15,T1の説明,user1,16,fx,SEQ001,SHOT001,
T2,2024/03/20,T2の説明,user2,24,animation,SEQ001,SHOT002,T1
T3,2024/03/25,T3の説明,user3,32,comp,SEQ002,SHOT001,"T1,T2"
```

## インポート手順
1. 管理画面の「CSVインポート」セクションに移動
2. 「CSVファイルを選択」ボタンをクリック
3. 上記の仕様に従って作成したCSVファイルを選択
4. 「インポート開始」ボタンをクリック
5. インポート結果を確認

## エラー処理
- 必須フィールドが欠落している場合はエラーとなります
- 日付形式が不正な場合はエラーとなります
- 依存タスクの指定が不正な場合はエラーとなります
- プロジェクト名やタスク名が重複する場合は、自動的に連番が付与されます
- プロジェクト名が空または100文字を超える場合はエラーとなります
- タスクの優先度が不正な値の場合はエラーとなります（HIGH, MEDIUM, LOWのみ有効）

## 制限事項
- 1回のインポートで処理できる最大行数: 1000行
- ファイルサイズの制限: 10MB
- サポートされる文字コード: UTF-8のみ
