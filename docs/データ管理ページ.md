# データ管理ページ

## 概要
このページでは、プロジェクト管理システムのデータを管理するための機能を提供します。

## 機能一覧

### 1. データのエクスポート
- 現在のデータベースの内容をJSON形式でエクスポート
- エクスポートされるデータ:
  - ユーザー情報
  - プロジェクト情報
  - タスク情報
  - イベント情報
  - グループ情報
  - ユーザーグループ関連情報

### 2. データのインポート
- JSON形式のデータをインポート
- インポート可能なデータ:
  - ユーザー情報
  - プロジェクト情報
  - タスク情報
  - イベント情報
  - グループ情報
  - ユーザーグループ関連情報

### 3. CSVファイルのアップロード
CSVファイルを使用してプロジェクトとタスクのデータを一括インポートできます。

#### ファイル形式
- 文字コード: UTF-8
- 区切り文字: カンマ (,)
- 改行コード: LF (Unix形式)

#### ファイル構造
CSVファイルは以下の2つのセクションで構成されます：

1. プロジェクト情報セクション
```
プロジェクト情報
プロジェクト名,開始日,終了日,説明
プロジェクトX,2024/03/01,2024/03/31,プロジェクトXの説明
```

2. タスク情報セクション
```
タスク情報
タスク名,期日,説明,担当者,コスト,タイプ,seqID,shotID,依存タスク
T1,2024/03/15,T1の説明,user1,16,fx,SEQ001,SHOT001,
T2,2024/03/20,T2の説明,user2,24,animation,SEQ001,SHOT002,T1
T3,2024/03/25,T3の説明,user3,32,comp,SEQ002,SHOT001,"T1,T2"
```

#### 注意事項
- 日付は「YYYY/MM/DD」形式で記述
- 複数の依存タスクは引用符（"）で囲む（例: `"T1,T2"`）
- プロジェクト名とタスク名が重複する場合は自動的に連番が付与
- プロジェクト名は1文字以上100文字以下である必要があります
- タスクの優先度が指定されていない場合は、デフォルトで「MEDIUM」が設定されます
- タスクのステータスは自動的に「TODO」に設定され、ステータス履歴が作成されます
- 詳細な仕様は[CSVファイルのアップロード仕様書](データ管理のCSVファイルのアップロード.md)を参照

### 4. モックデータの管理
- モックデータのインポート/エクスポート
- テスト用データの管理

## アクセス制限
- 管理者権限を持つユーザーのみが利用可能
- 一般ユーザーはアクセス不可

## 注意事項
- データのインポート/エクスポートは慎重に行う
- 本番環境での操作は特に注意が必要
- 定期的なバックアップを推奨
- CSVインポート時のエラーはログに記録され、詳細なエラーメッセージが表示されます
